# 验收标准

## P0 前台优化验收

### 1. 延迟验收

| 场景 | 改造前基线 | 验收标准 |
|------|-----------|---------|
| 普通闲聊/问候 | 5-8s | ≤ 4s |
| 商品知识查询 | 6-9s | ≤ 5s |
| 客户资料提供 | 5-8s | ≤ 4s |
| 高风险/金融/售后 | 7-10s | ≤ 6s |
| 意图识别超时降级 | 5-8s | ≤ 4s |

**测试方法**：
- 发送 20 条测试消息（覆盖上述 5 种场景各 4 条）
- 记录 `listen_and_reply.py --once` 的单次执行耗时
- 取中位数，与基线对比

### 2. 意图识别质量验收

- 意图分类准确率 ≥ 90%（人工抽查 50 条）
- 超时降级率 ≤ 5%（查看日志中 `source: "keyword_fallback"` 的比例）

### 3. 拟人化质量验收

- 人工抽查 30 条回复，机械化/模板化感知率 ≤ 20%
- 同一场景连续 3 次回复，句式重复率 ≤ 30%

### 4. 稳定性验收

- 连续运行 24 小时，listener 进程不崩溃
- `rt.status()` 始终准确反映进程状态
- 无重复 listener 进程（`_already_running` 正常工作）

## P1 后台 Worker 验收

### 1. 前台延迟进一步降低

| 场景 | P0 基线 | P1 验收标准 |
|------|---------|------------|
| 普通闲聊/问候 | 4s | ≤ 3.5s |
| 商品知识查询 | 5s | ≤ 4s |
| 客户资料提供 | 4s | ≤ 3s |
| 高风险/金融/售后 | 6s | ≤ 5s |

### 2. 后台任务执行验收

- 后台 worker 进程稳定运行，内存泄漏检测通过（24 小时监控）
- 队列中 pending 任务不堆积超过 100 条
- 任务完成率 ≥ 95%（查看 `succeeded / (succeeded + failed)`）

### 3. 数据一致性验收

- 客户数据最终仍正确写入 Excel（可能延迟 5-30 秒）
- RAG 经验最终仍正确吸收（可能延迟 1-5 分钟）
- 原始消息记录无丢失

### 4. 生命周期管理验收

- `rt.start()` 同时启动 listener + worker
- `rt.stop()` 同时停止 listener + worker
- `rt.status()` 显示 `worker_running` 和 `queue_pending_count`

## 回归测试清单

改造完成后必须跑以下回归测试：

1. **基本对话流程**：问候 → 商品咨询 → 价格询问 → 转人工
2. **客户资料采集**：提供姓名 → 提供电话 → 确认入库
3. **安全边界测试**：要求最低价 → 要求账期 → 要求虚开发票（均应转人工或拒绝）
4. **RAG 经验命中**：发送已有经验覆盖的问题（应走 RAG 回复）
5. **多轮上下文**：连续追问"刚才那台""这个车"（应正确解析指代）
6. **速率限制**：快速发送多条消息（应触发 backoff）
