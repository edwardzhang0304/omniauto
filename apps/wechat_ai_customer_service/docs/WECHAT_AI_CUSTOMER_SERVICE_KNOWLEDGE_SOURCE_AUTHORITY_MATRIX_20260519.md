# 知识来源权威矩阵 2026-05-19

## 原则

一句话原则：能观察不等于能学习，能学习不等于能检索，能检索也不等于能授权事实。

## 来源分级

| 来源 | 默认可信度 | 能否直接学习 | 能否直接检索 | 是否能授权事实 | 备注 |
| --- | --- | --- | --- | --- | --- |
| 商品主数据手动导入 | 高 | 否 | 由商品库专用链路引用 | 是 | 车型、价格、库存、配置只能走商品主数据 |
| 正式知识库人工规则 | 高 | 否 | 是 | 可授权规则边界 | 只存稳定政策、流程、风险边界 |
| 清洗后真实聊天 JSONL | 中 | 是，进入 RAG 经验 | 审核后才可检索 | 否 | 用于风格、话术、场景经验 |
| RAG 经验人工保留 | 中高 | 已学习 | 是 | 否 | 只能提供经验参考，不能改商品事实 |
| AI 智能记录员选中群 | 中 | 是，先 review-only | 审核后才可检索 | 否 | 需要用户明确选择数据源 |
| 微信客服监听消息 | 低 | 默认否 | 否 | 否 | 只作为当前回复上下文和审计 |
| 文件传输助手 | 测试通道 | 否 | 否 | 否 | 永远不学习 |
| AI/机器人自回复 | 禁止 | 否 | 否 | 否 | 防止模型回声污染 |
| 测试批次/演示批次 | 禁止 | 否 | 否 | 否 | 含 REALWX/LIVEFLOW/GENLIVE/LLMSYN/DBG/CHEJIN_DEMO 等 marker |
| 商品资料从 RAG 晋升 | 禁止 | 否 | 否 | 否 | 商品只允许手工维护 |

## 开发约束

1. 新增学习入口时，必须显式写清 `source_module`、`conversation_type`、`source_type`、`category`。
2. 新增 RAG chunk 前，必须通过 `rag_chunk_is_retrievable()`。
3. 新增 raw message batch 前，必须按消息级 `message_learning_exclusion_reason()` 过滤。
4. customer-service 实时监听不得默认开启学习。
5. 新增 live test marker 时，必须同步加入 `TEST_MARKER_RE`。
6. 任何商品形态内容不得从聊天经验、RAG 候选或知识升级链路写入商品主数据。

## 常见误区

1. “客户问过一次”不是知识。
2. “AI 回过一次”更不是知识。
3. “测试中出现过的好回复”不能直接变成知识，应先整理成规则或经验样本。
4. 商品事实不能靠聊天归纳，必须来自商品库。
5. 实盘话术适配器只改变说法，不改变事实边界。
