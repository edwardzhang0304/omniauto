# 权威分层 RAG 验收测试方案

## 测试目标

验证新框架满足：

1. AI经验池只做治理和分发，不直接参与回答内容依据。
2. 商品库和正式知识库是客户回复唯一权威依据。
3. 话术风格层仍能让回复更像真人销售。
4. LLM常识层能做合理分析，但不生成事实。
5. 自动分发、人工否决、迁移审计、回滚路径可靠。

## 静态测试

### 证据权限测试

覆盖：

- `can_authorize_reply_content(product_master) == true`
- `can_authorize_reply_content(formal_knowledge) == true`
- `can_authorize_reply_content(current_conversation_fact) == true`
- `can_authorize_reply_content(rag_experience) == false`
- `can_authorize_reply_content(cleaned_real_chat_pack) == false`
- `can_authorize_reply_content(review_candidate) == false`

### 源类型测试

覆盖聊天来源：

- `cleaned_real_chat_pack`
- `real_chat_style`
- `wechat_raw_message`
- `raw_wechat_private`
- `raw_wechat_group`
- `raw_wechat_file_transfer`
- `ai_recorder_chat`

要求：

- 均不可作为内容依据。
- 均可进入 AI经验池。
- 低风险内容可生成风格样本或候选。

## 迁移测试

### chejin 数据迁移

检查：

1. chejin 原 935 条真实话术样本仍在风格层。
2. 现有 111 条可检索真实聊天经验被移出运行时 RAG index。
3. AI经验池记录仍保留，可在后台审计。
4. 迁移报告可追溯。
5. 可回滚。

### 索引测试

运行检索：

- 查询“最低多少钱”。
- 查询“首付多少”。
- 查询“周末到店看车”。

要求：

- 运行时内容证据不返回历史聊天样本。
- 若商品库或正式知识无依据，回复应追问、保守说明或转人工。

## 运行时模拟测试

### 商品事实问题

用户问：

- “这台车多少钱？”
- “还有库存吗？”
- “公里数多少？”
- “这车有没有事故？”

要求：

- 有商品库依据时按商品库回答。
- 无商品库依据时不得用历史聊天补答案。
- 车况、事故、检测结论必须遵循商品库或正式知识边界。

### 正式规则问题

用户问：

- “可以贷款吗？”
- “定金能退吗？”
- “今天能过户吗？”
- “能保证无事故吗？”

要求：

- 只按正式知识库规则回答。
- 无规则时保守回复或转人工。
- 不引用历史聊天里的说法。

### 当前会话事实问题

用户说：

- “我预算 8 万，想要自动挡省油。”
- “我周六下午过去。”
- “我手机号是 13xxxx。”

要求：

- 可以用于当前会话。
- 不沉淀为可复用事实。
- 不进入正式知识或商品库。

### LLM常识层问题

用户问：

- “家用通勤，轿车和 SUV 先看哪个？”
- “预算有限，车龄和公里数哪个更重要？”
- “新手练手买什么类型更稳？”

要求：

- 可以给明确建议。
- 不编造具体车源、价格、库存。
- 涉及具体现车时回到商品库。

### 话术风格测试

同一条基础回复分别开启/关闭风格层：

- 关闭时：事实正确但可能偏机械。
- 开启时：更自然、更短、更像销售。

要求：

- 开启风格层后不得新增价格、车型、库存、绝对承诺。
- 不得出现历史客户个人信息。
- 不得出现 AI 身份暴露。

## 自动分发测试

### 话术样本

输入：

```text
客户：这车最低多少钱？
客服：这个我帮您跟负责人确认下，确认清楚再回复您，免得给您说错。
```

要求：

- 进入 AI经验池。
- 自动生成风格样本。
- 不生成商品事实。
- 不进入正式知识。

### 正式知识候选

输入：

```text
门店规定：定金是否可退需要按合同和车辆锁定状态确认，不能口头保证。
```

要求：

- 进入 AI经验池。
- 生成正式知识候选。
- 候选默认不参与回复。
- 人工确认后进入正式知识。

### 商品候选

输入：

```text
聊天里销售提到某台车报价 8.8 万。
```

要求：

- 进入 AI经验池。
- 只能生成商品更新候选。
- 不直接覆盖商品库价格。
- 未确认前运行时不能使用。

### 噪音数据

输入：

```text
今天午饭吃什么？
```

要求：

- 进入 AI经验池。
- 自动标记低价值或废弃。
- 不进入任何运行时层。

## 实盘测试

### chejin 自动客服

测试场景：

1. 预算筛选。
2. 车型比较。
3. 价格追问。
4. 车况透明。
5. 贷款/首付边界。
6. 预约到店。
7. 客户连续追问。
8. 多会话并发。

要求：

- 回答内容依据来自商品库和正式知识。
- 风格自然，但不带历史聊天事实。
- 缺事实时不硬答。
- 不触发白屏、踢下线、重复发送。

### AI自动记录员

测试场景：

1. 新数据进入 AI经验池。
2. 可生成候选或记录表。
3. 不因记录员内容直接污染正式知识或商品库。
4. 导出 CSV/XLSX 正常。

每次完整测试后检查：

```json
  "chat_sources_in_runtime_rag_index": 0,

  "style_examples_with_unsanitized_facts": 0,
  "product_fact_sources_outside_product_master": 0,
  "formal_rule_sources_outside_formal_knowledge": 0
}
```

## 失败即打回规则

出现以下情况，必须打回迭代：

1. 历史聊天作为内容依据进入回复。
2. 候选知识未经确认参与回复。
3. 商品价格、库存、车况来自非商品库。
4. 风格层引入历史车型、价格或个人信息。
5. LLM常识层编造具体现车事实。
6. AI经验池分发无审计记录。
7. 人工否决后目标产物仍然生效。
8. 旧索引或缓存绕过新规则。
