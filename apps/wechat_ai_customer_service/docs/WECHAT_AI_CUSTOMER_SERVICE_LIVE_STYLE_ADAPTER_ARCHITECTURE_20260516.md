# 微信自动客服实盘话术风格适配器架构说明

## 目标

实盘话术风格适配器用于把各账号真实客服聊天记录中的表达方式沉淀为可复用的回复风格层。

它是账号通用能力，不绑定二手车、家电或任何单一行业。每个账号使用自己的实盘样本，模块逻辑保持一致。

## 位置

推荐链路：

1. 客户消息进入意图识别。
2. 商品库、政策库、正式知识、RAG经验提供事实和业务依据。
3. 实时规则、RAG回复、LLM合成生成基础回复。
4. 实盘话术风格适配器只调整表达方式。
5. 安全守卫和防AI暴露策略做最终兜底。
6. 发送给客户。

## 边界

适配器可以学习：

- 称呼方式，但必须遵守公共底层规则：只在开场寒暄和新话题切入时有选择地使用；连续对话中不要每条都加；男性称呼可在“姓哥/哥/老板”之间自然轮换，女性称呼可用“姓姐/姐”；不能把文件传输助手、群名、门店名、测试名等非真人会话名拆成姓氏称呼。
- 句子长短。
- 接住客户情绪的方式。
- 追问预算、用途、时间、资料的顺序。
- 轻销售推进方式。
- 边界问题的真人式缓冲话术。
- 避免机械重复的表达变体。

适配器不能学习或生成：

- 商品价格。
- 库存状态。
- 具体优惠。
- 合同、发票、售后承诺。
- 事故、水泡、火烧等确定性车况承诺。
- 客户隐私。
- 旧话术里的显式“转人工/人工客服”表达。
- 任何会暴露 AI 身份的表达。

## 模块

`style_memory_store.py`

- 从当前证据包、正式聊天样本、RAG实盘片段中提取可借鉴的客服表达。
- 对样本做去重、清洗、风险标记和相似度排序。
- 支持未来扩展为租户级 `style_memory` 独立索引。

`reply_style_adapter.py`

- 接收基础回复和风格样本。
- 根据当前回复来源判断是否适配。
- 在本地快速模式下做低成本表达调整。
- 做事实不漂移、安全词和防AI暴露检查。
- 输出审计信息，便于回看是否生效。

## 配置

推荐租户配置：

```json
{
  "reply_style_adapter": {
    "enabled": true,
    "mode": "fast_local",
    "apply_to_realtime": true,
    "apply_to_rag": true,
    "apply_to_llm": true,
    "apply_to_rule": true,
    "apply_to_handoff": true,
    "identity_guard_aware": true,
    "avoid_repetition": true,
    "max_examples": 3,
    "min_similarity": 0.08,
    "include_rag_style_hits": true,
    "max_reply_chars": 520,
    "shadow_mode": false
  }
}
```

## 运行模式

当前落地优先使用 `fast_local`。

- 不额外调用大模型。
- 不增加前台 token 消耗。
- 只做安全的短语级、结构级表达适配。
- 失败时自动回退原回复。

未来可以扩展 `llm_rewrite`。

- 只在低风险、证据充分、配置允许时开启。
- 输入基础回复和风格样本。
- 输出仍必须通过事实不漂移和身份守卫。

## 多账号复用

模块根据当前租户读取风格样本。

- `chejin` 账号会学习二手车实盘话术。
- 家电账号会学习家电客服话术。
- 新账号没有足够样本时，适配器自动保守降级。

行业差异由样本数据体现，模块本身不写死行业规则。
