# WeChat AI 客服标准工作流验收清单

> 适用范围：任意租户、任意行业的微信 AI 客服数据学习与发布流程。

## A. 数据治理验收

- [ ] 清洗后模板文件可解析为 UTF-8 JSONL
- [ ] 模板 ID 全量唯一
- [ ] `service_reply` 必填完整，无空值
- [ ] 手机号泄漏计数为 0
- [ ] 真实姓名泄漏计数为 0
- [ ] 元数据污染词（风险等级/规则名称等）泄漏计数为 0
- [ ] 低信息短句占比符合阈值（建议 <= 8%）
- [ ] 生成清洗报告与拒绝原因分布报告

## B. Schema 与入库验收

- [ ] 通过 `knowledge_intake.evaluate_intake_item` 全量校验
- [ ] `needs_more_info` 数量为 0
- [ ] runtime 字段有默认值（`allow_auto_reply/requires_handoff/risk_level`）
- [ ] `source.batch_token` 与版本号一致
- [ ] 导入前 dry-run 报告已归档

## C. 检索与编排验收

- [ ] 事实轨（products/policies/product_scoped）命中正常
- [ ] 风格轨（chats）仅影响表达，不覆盖事实字段
- [ ] 经验轨（rag_experience）不绕过高风险边界
- [ ] 高风险场景命中后可触发 handoff
- [ ] 证据包可追溯命中来源与轨道

## D. 回复质量验收

- [ ] 事实一致率不低于基线
- [ ] 违规承诺率不高于基线
- [ ] 转人工准确率不低于基线
- [ ] 客户续聊率不低于基线
- [ ] 语气自然度主观抽检通过（业务侧抽样）

## E. 发布与回滚验收

- [ ] 发布版本号已生成
- [ ] 变更摘要已登记
- [ ] 回滚点可用并已演练
- [ ] 灰度结果报告已归档
- [ ] 全量发布审批记录完整

## F. 审计与追踪验收

- [ ] 原始数据批次有只读归档
- [ ] 清洗、导入、评估、发布均有审计日志
- [ ] 线上异常可追溯到具体版本与批次

## G. 多行业可复用验收

- [ ] 不改主流程，仅替换行业词典即可跑通
- [ ] 新行业可复用相同验收清单
- [ ] 行业插件配置与主流程边界清晰分离
