# LLM 语义质量门 V2 文档索引

## 背景

当前微信智能客服已经切到 Brain First 主线：由 LLM 客服大脑理解客户消息、结合商品库与正式知识库规划回答，再经过安全 guard 与最终可见润色后发送。

最近实盘和模拟测试暴露出一个新的结构性问题：Brain 大脑本身很多时候已经理解正确，但后置质量门仍带有较强的静态规则特征，容易把“语义上合理、但没有命中固定表达”的回复误判为失败，导致答非所问、重复追问、错误转人工，或者退回到更机械的回复路径。

因此本轮设计目标不是继续堆更多硬编码规则，而是把质量门升级为：

```text
硬边界校验 + LLM 语义审稿 + Brain 修复回路 + 最终可见校验/微润色
```

核心原则：

- Brain 是正常客服对话的主思考者。
- 硬边界 guard 负责不可越权的事实、安全、会话隔离、身份暴露等约束。
- LLM 语义审稿只判断“这句回复是不是像一个合格真人客服在当前上下文里的合理回答”。
- LLM 语义审稿不能授权商品事实、政策承诺、价格库存，也不能覆盖硬边界 guard。
- 所有客户可见文本仍必须经过最终可见层；Brain 来源回复默认进入校验/微润色模式，不允许最终润色重新决策或改写语义。

## 推荐实施顺序

1. 先新增语义审稿模块，shadow 模式只记录不拦截。
2. 接入离线合同测试，验证不会放过硬事实错误。
3. 将部分软规则从静态拦截迁移为 LLM 语义审稿建议。
4. 开启 suspicious_only 模式，只对可疑回复调用语义审稿。
5. 实盘低风险测试通过后，再考虑 always 模式或更严格灰度。
