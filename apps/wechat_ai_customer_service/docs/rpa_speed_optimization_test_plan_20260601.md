# 微信自动客服 RPA 响应加速测试计划（2026-06-01）

## 1. 测试目标
验证“freshness 轻量化 + 多段发送验证优化 + 拆段阈值调优”后：
1. 不降低回复质量与安全性。
2. 提升响应速度，重点降低 `send_stage`。

## 2. 测试范围
### 2.1 功能正确性
- freshness 判定正确（新消息到达时不发旧回复）。
- 多段发送语义完整、顺序正确、无错会话。
- 单段/多段、短句/长句均可稳定发送。

### 2.2 稳定性与鲁棒性
- 多会话并发调度。
- 目标会话切换场景（文件传输助手/许聪/新数据测试）。
- 发送失败重试与降级路径。

### 2.3 性能
- 对比基线的 `capture / llm / send_stage / end_to_end` 指标。
- 分析是否满足“提速且质量不降”。

## 3. 用例矩阵
### T1 离线逻辑
1. 预览无未读 -> freshness 快速放行。
2. 预览有未读 -> freshness stale。
3. 触发严检间隔 -> 执行严格 freshness。
4. 多段发送 `verify_each_segment=false` -> 仅末段强验证。
5. 多段发送 `verify_each_segment=true` -> 每段强验证兼容。

### T2 模拟并发
1. 两会话轮询 + 连续追问。
2. 三会话并发 + 不同长度回复混合。
3. 检查是否出现错位发送、漏回、重复回。

### T3 实盘
1. 文件传输助手连续对话（短句、长句、追问、话题切换）。
2. 多会话开启后随机单会话触发新消息，确认不会错发。
3. 记录耗时与事件日志，形成对比报告。

## 4. 指标与阈值
### 4.1 正确性
- `send_failed`、`reply_stale`、`target_not_confirmed` 仅在合理场景出现。
- 无“回答发错会话”。

### 4.2 质量
- 无新增“答非所问/边界误答/机械化明显上升”。
- 长回复仍可自然拆分，语义完整。

### 4.3 性能
- 与基线相比：
1. `send_stage p50` 明显下降。
2. `end_to_end p50` 明显下降。
3. `llm` 质量不降前提下不显著劣化。

## 5. 结果产物
- 静态与离线测试输出（json）。
- 模拟/实盘运行日志与关键事件摘要。
- 前后指标对比 JSON（baseline vs after）。
