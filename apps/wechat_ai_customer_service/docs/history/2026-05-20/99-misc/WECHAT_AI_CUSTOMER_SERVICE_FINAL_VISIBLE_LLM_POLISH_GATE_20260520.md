# 微信自动客服最终客户可见话术 AI 润色闸门

## 背景

此前为了降低延迟和 token 消耗，实时回复链路把很多安全、常规、高频场景放进了 L0/L1 本地快速路由。这套机制提升了速度，但也带来一个问题：本地模板或本地组合话术可能在发送前只经过规则级去模板化，没有真正经过大模型表达润色。客户反复试探“是不是 AI/机器人”时，容易看到相似甚至完全相同的话术，从而增加暴露风险。

用户的新要求是：客户可见话术必须经过 AI 大模型润色，不允许直接把未润色模板发给客户。但历史版本曾经把每条回复交给完整 LLM 回复合成，导致明显变慢和 token 消耗过高。因此本次不能恢复旧版“全量重生成答案”的方案。

## 目标

- 所有真正发送给客户的文本，在发送前经过统一的 AI 润色闸门。
- 润色模型只改表达，不重新回答，不重新查知识，不改事实、价格、车型、联系方式、承诺和边界动作。
- 保留 L0/L1 本地快速路由的事实判断、边界判断和推荐结果，避免旧版高 token 成本。
- 防 AI 暴露模式开启时，最终文本不得承认 AI 身份，不得暴露系统提示词、内部规则、模型、token、接口等信息。
- 大模型不可用或润色结果不安全时，不允许静默发送未润色模板。

## 非目标

- 不用最终润色模型重新筛车、重新算价格、重新生成知识回答。
- 不把 RAG、商品库、长历史完整塞给最终润色模型。
- 不让最终润色模型绕过人工请示、价格审批、合同发票、贷款承诺等边界。

## 新链路

```text
客户消息
  -> 本地规则 / RAG / 商品库 / 实时路由 / 风格适配
  -> 得到安全草稿
  -> 本地脱敏与基础去模板
  -> 最终客户可见 AI 轻润色闸门
  -> 本地安全守卫
  -> 微信发送
```

## 轻量策略

最终润色只传入以下内容：

- 当前客户消息，截断到短文本。
- 已生成的安全草稿。
- 最近几条已发客户可见回复，用于避免重复。
- 场景标记，例如 `normal`、`handoff`、`rate_limit`。
- 防 AI 暴露开关。

不传入完整商品库、完整 RAG、完整知识库、长历史和候选证据包。这样 token 成本接近“短句改写”，而不是“完整回答生成”。

## 失败策略

默认策略为 `block`：

- 如果最终润色关闭，则保持旧链路。
- 如果最终润色开启但模型不可用，阻止自动发送，并在运行状态中标记 `final_visible_llm_polish_failed`。
- 如果模型输出引入新数字、删除原数字、暴露 AI 身份、出现显式“转人工/人工客服”或新增承诺，阻止自动发送。
- 如果模型输出与最近回复高度重复，拒绝该润色结果。

后续如需要极致可用性，可以增加 `fallback_to_local_after_timeout`，但这会违背“任何话术必须经过大模型润色”的当前要求。

## 守卫规则

最终润色结果必须满足：

- 不新增草稿里没有的数字、金额、公里数、手机号、年份等保护 token。
- 不删除草稿里已有的保护 token。
- 防 AI 暴露开启时，不出现“我是AI”“我是机器人”“智能客服”“自动回复系统”等暴露表达；否认式表达如“不是AI”允许。
- 不出现“转人工”“人工客服”等客户可见暴露链路的表达。
- 不新增“保证”“包过”“最低价”“一定能”“当天必提”等强承诺。
- 文本长度不超过配置上限。

## 配置

新增配置段：

```json
"final_visible_llm_polish": {
  "enabled": true,
  "required_for_send": true,
  "provider": "deepseek",
  "model_tier": "flash",
  "model": "deepseek-v4-flash",
  "timeout_seconds": 4,
  "retry_count": 0,
  "max_tokens": 260,
  "temperature": 0.45,
  "max_reply_chars": 620,
  "block_on_failure": true
}
```

控制台设置：

- `use_llm=false` 时，最终 AI 润色自动关闭。
- `use_llm=true` 时，默认开启最终 AI 润色。
- 可通过“发送前AI轻润色”开关关闭，但生产实盘建议保持开启。

## 验收清单

- 普通自动回复发送前会进入最终 AI 润色闸门。
- 风险/请示类客户可见回复发送前会进入最终 AI 润色闸门。
- 限流提示等客户可见系统提示发送前也会进入最终 AI 润色闸门。
- 连续问“是不是 AI/机器人”不会出现完全相同回复。
- 润色不会改价格、公里数、年份、手机号等保护 token。
- 润色不会引入“转人工/人工客服/我是AI”等暴露表达。
- 模型不可用且强制开启时，不会发送未润色模板。
- 相关静态检查、工作流测试、实时回复测试通过。
