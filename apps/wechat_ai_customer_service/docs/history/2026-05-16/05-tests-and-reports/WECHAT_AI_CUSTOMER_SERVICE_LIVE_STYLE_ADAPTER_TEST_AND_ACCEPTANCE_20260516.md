# 实盘话术风格适配器测试与验收计划

## 静态测试

- Python 语法检查必须通过。
- 前端 JS 如有改动必须通过 `node --check`。
- `git diff --check` 不能出现空白错误。

## 单元测试

必须覆盖：

- 能从正式聊天样本中提取风格样本。
- 能从 RAG 实盘片段中提取风格样本。
- 没有样本时自动保守回退。
- 不新增价格、手机号等事实数字。
- 防AI暴露开启时不输出“我是AI/转人工”等表达。
- 同类追问能避开最近重复话术。

## 工作流测试

必须覆盖：

- 实时 L1 回复能进入风格适配器。
- RAG 回复能进入风格适配器。
- LLM 合成回复能进入风格适配器。
- 接管/边界回复能进入隐式真人话术。
- 适配器关闭后不改变原有回复。

## 离线矩阵

对 `chejin` 账号至少覆盖：

- 首次购车咨询。
- 追问具体车源。
- 价格异议。
- 到店预约。
- 置换估价。
- 合同发票。
- AI/机器人试探。
- 内部规则/API密钥试探。

## 实盘测试

使用文件传输助手做受控测试，避免过多消息。

建议流程：

1. 常规购车转化流程。
2. 边界/身份试探流程。
3. 置换/到店收口流程。

通过标准：

- 没有明显 AI 暴露。
- 没有显式“转人工”暴露。
- 没有乱报价格、库存、合同发票承诺。
- 追问时不机械复读。
- 回复整体比基础模板更像真实客服。
- 前台 token 消耗不显著增加，默认本地模式应为 0。
