# 微信自动客服 RPA 风险收束与沙盒双号测试执行文档（2026-05-28）

## 目标

把前两轮要求记录的内容转成可执行门禁、测试与实盘流程：

1. 先优化普通实盘路径的 RPA 风险控制，重点防止错会话、异常窗口继续操作、机械回滚和高频探测。
2. 完成静态测试、聚焦测试、全量非实盘测试和低频文件传输助手实盘回归。
3. 普通路径通过后，再优化沙盒双号测试相关门禁。
4. 再进行沙盒微信模拟真人测试；若任一账号被踢出、登录态异常、白屏或目标不确定，立即停机并转人工通知。

## 约束

- 不伪造硬件、设备、网络、客户端指纹。
- 不绕过微信登录、安全验证或风控逻辑。
- 不恢复 wxauto4 优先级；wxauto4 只作为技术储备。
- 不做三会话高压刷屏；实盘只做低频、可停、可恢复的受控验证。
- 测试刷屏只允许在明确边界测试中使用，平时不允许一次性粘贴大段内容。

## 第一阶段：普通 RPA 风险优化

### P0 硬门禁

| 风险 | 优化动作 | 验收 |
|---|---|---|
| 运行中目标扩展到非白名单会话 | 监听循环检查所有事件目标；只要出现非 `live_safety_guard.allowed_targets` 目标，立即停机并创建转人工 case | 聚焦测试覆盖非白名单目标被阻断 |
| 白屏/辅助壳/异常窗口后继续探测 | 将 `blank_render_detected`、`auxiliary_shell_window_detected`、异常窗口几何等纳入 transport risk 硬停 | 聚焦测试覆盖异常窗口直接 stop |
| 历史回滚找不到锚点仍回复 | 保持当前锚点机制：找不到锚点则阻断自动回复 | 既有 anchor backfill 测试继续通过 |
| 重复发送前输入确认失败 | 保持阈值熔断与冷却 | 既有 transport risk 测试继续通过 |

### P1 低扰动策略

- `live_safety_guard` 开启时强制单目标低风险扫描。
- `respond_all_unread_sessions` 必须关闭。
- `history_backfill` 仅允许锚点驱动模式，禁止固定次数盲滚。
- 被动探针不做窗口归一化、不自动进入快捷登录、不发送消息。
- 最小化/离屏/白屏时只提示人工处理，不自动恢复窗口后继续发消息。

## 第二阶段：普通路径测试

1. 文档校验：`git diff --check`。
2. Python 静态编译：触达文件 `py_compile`。
3. 聚焦测试：
   - `run_realtime_reply_optimization_checks.py`
   - `run_workflow_logic_checks.py`
   - `run_wechat_win32_ocr_compat_checks.py`
   - `run_customer_service_multi_session_scheduler_checks.py`
4. 全量非实盘矩阵：遍历 `apps/wechat_ai_customer_service/tests/run_*_checks.py`，跳过真实微信发送脚本。
5. RPA 只读验收：`run_rpa_acceptance_report.py --tenant-id chejin --wechat-probe passive`。
6. 实盘低频回归：文件传输助手 anchor backfill 回归，验证：
   - 当前屏幕有锚点时不滚动。
   - 当前屏幕无锚点时，小步上翻，找到锚点即停。
   - 找不到锚点时不发。
   - 无白屏、无踢出、无错会话。

## 第三阶段：沙盒双号测试优化

### 前置门禁

| 门禁 | 要求 |
|---|---|
| 窗口隔离 | 宿主微信固定左侧，沙盒微信固定右侧，不重叠 |
| 登录态 | 两端均已登录，且无白屏、二维码、快捷登录页、加载空白 |
| 目标白名单 | 宿主 AI 客服只处理 `许聪`；沙盒只允许发给 `Meta_xc` 和 `文件传输助手` |
| 操作节奏 | 一端发送后等待另一端完成，不双端高频切换 |
| 内容自然度 | 使用真实咨询语气，避免纯数字、时间戳、重复短句 |
| 异常策略 | 目标、输入框、发送状态、锚点、窗口任一不确定即停 |

### 沙盒测试顺序

1. 沙盒只读/窗口识别，不发送。
2. 沙盒给文件传输助手发 1 条自然短句，验证沙盒端可控。
3. 沙盒 `许聪` 给宿主 `Meta_xc` 发 1 条自然咨询。
4. 宿主 AI 客服识别 `许聪` 会话并回复。
5. 冷却 2 到 3 分钟，再进行第二轮。
6. 3 到 5 轮通过后，再测长句、分多条连续补充、窗口大小恢复。

## 停止条件

出现以下任一情况立即停止，不再尝试自动恢复：

- 任一微信账号被踢出、出现二维码、快捷登录页或登录异常。
- 微信白屏、辅助壳窗口、异常空白、窗口句柄无效。
- 当前目标不是本轮白名单目标。
- 输入框、发送按钮、聊天标题无法确认。
- 锚点追溯找不到边界且存在新消息缺口。
- 出现操作频繁、账号安全、登录环境异常等安全提示。

## 验收标准

- 所有静态与全量非实盘测试通过。
- 普通文件传输助手低频实盘通过。
- 沙盒相关门禁测试通过。
- 沙盒双号实盘若执行，必须在未踢出、未白屏、未错会话的前提下完成设定轮次。
- 停机后无 listener、operator guard、recorder、live test 残留进程。
- 完成或阻断时发送 ServerChan 通知。
