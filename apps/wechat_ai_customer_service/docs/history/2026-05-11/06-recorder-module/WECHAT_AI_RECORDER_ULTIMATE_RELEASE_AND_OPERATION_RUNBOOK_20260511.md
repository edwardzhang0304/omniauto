# 微信AI智能记录员终极上线与运维手册（2026-05-11）

## 1. 目标
提供从预发验证、灰度上线、正式发布到异常回滚的标准执行流程，并覆盖管理员与业务用户日常操作。

## 2. 上线前准备
1. 文档冻结：PRD、架构、API、LLM协议、测试报告均有版本。
2. 模块准备：`order_sheet_lab_v1` 已注册并设为可用。
3. 账号绑定：目标账号已绑定模块。
4. 数据准备：测试数据与验收样本已验证可读取。
5. 监控准备：任务状态、耗时、失败率、模型错误率可观测。

## 3. 发布步骤
1. 部署后端服务改动。
2. 部署管理后台前端改动。
3. 执行配置迁移（模块默认参数、日期模式、上限参数）。
4. 重启相关服务并检查健康状态。
5. 跑冒烟测试：
- 创建“导出所有记录（结构化）”任务。
- 按日快捷导出。
- 按周快捷导出。
- 下载并检查表头格式。

## 4. 灰度策略
1. 第一阶段：仅开放给测试账号（如 test2）。
2. 第二阶段：开放给单客户真实场景。
3. 第三阶段：扩大到多个账号。
4. 每阶段观察：
- 完成率。
- 平均耗时。
- 低置信度比例。
- 用户反馈问题数。

## 5. 回滚策略
1. 模块级回滚：账号绑定回退到上一稳定模块版本。
2. 配置级回滚：恢复默认参数（如 `max_messages`、阈值）。
3. 服务级回滚：回退到上一版本构建。
4. 回滚原则：
- 不删除历史任务记录。
- 不覆盖已导出的历史文件。
- 回滚后仅影响新建任务。

## 6. 运行监控与告警
1. 任务堆积告警：`queued` 超时。
2. 处理卡顿告警：单阶段耗时异常。
3. 模型异常告警：`llm_upstream_error` 激增。
4. 导出失败告警：`export_generation_failed` 超阈值。
5. 质量告警：低置信度占比连续升高。

## 7. 日常运维SOP（管理员）
1. 检查模块状态与版本。
2. 检查账号绑定是否正确。
3. 检查任务队列与失败任务。
4. 对失败任务执行重试或复核。
5. 每周输出质量复盘。

## 8. 日常操作SOP（业务用户）
1. 在“AI智能记录员”进入导出区域。
2. 普通导出：点击“导出所有记录（结构化）”。
3. 高级导出：展开高级筛选，选日期范围后执行导出。
4. 快捷导出：按日/按周/按月。
5. 查看进度：观察当前阶段和处理条数。
6. 完成后下载 Excel 和 JSON（用于复核）。

## 9. 故障排查指南
1. 现象：长时间停留“排队中”。
排查：队列是否阻塞、工作进程是否在线、并发限制是否过低。
2. 现象：停留“处理中”超过阈值。
排查：卡在 `extracting` 还是 `reviewing`，检查模型调用日志。
3. 现象：导出结果明显缺字段。
排查：模块是否绑定正确、日期范围是否过窄、过滤条件是否误配。
4. 现象：准确率波动。
排查：样本分布变化、提示词版本变化、是否触发降级策略。

## 10. 与微信推送联动（长任务）
1. 长任务完成必须触发微信通知（成功/失败都通知）。
2. 通知内容至少包含：
- 任务名称
- 状态
- 耗时
- 下载路径或失败摘要
3. 若推送失败，必须记录重试并在后台可见。

## 11. 安全与审计
1. 导出任务操作写审计日志（创建者、时间、参数摘要）。
2. 模块绑定操作写审计日志（变更前后）。
3. 敏感配置变更须记录操作者与原因。
4. 关键故障处理保留操作记录。

## 12. 上线后7天观察清单
1. 完成率是否稳定 >= 99%。
2. 平均耗时是否在可接受范围。
3. 低置信度比例是否下降。
4. 主要错误类型是否集中且可修复。
5. 是否需要发布 `order_sheet_lab_v1.1`。
