# 微信AI智能记录员终极优化开发清单（2026-05-11）

## 1. 目标
将终极方案拆成可执行工程包，明确任务优先级、依赖关系、交付物和完成定义（DoD）。

## 2. 开发包分解
## 2.1 Package A：通用底座补完（P0）
任务：
1. 结构化导出任务状态机标准化（含阶段进度）。
2. 默认上限 `max_messages=10000`，不足则全量输出。
3. 日期范围筛选与快捷导出（日/周/月）通用实现。
4. 前端自动轮询和阶段进度展示（移除手工刷新依赖）。
5. “微信智能客服”与“AI智能记录员”双开关隔离。

交付物：
1. 后端服务改造。
2. 管理后台交互改造。
3. 回归脚本更新。

DoD：
1. 按日、按周导出可用。
2. 任务从入队到完成全程可见阶段。
3. 不影响现有客服功能链路。

## 2.2 Package B：模块化能力治理（P0）
任务：
1. 模块注册与版本管理。
2. 账号级模块绑定与租户默认兜底。
3. 导出任务记录模块快照（ID+版本+关键参数）。
4. 失败回退到稳定模块机制。

交付物：
1. 模块服务接口。
2. 管理端配置入口。
3. 审计字段与日志。

DoD：
1. admin可对账号切换模块。
2. 切换后新任务立即生效。
3. 历史任务不受切换影响。

## 2.3 Package C：LLM主导抽取链路（P0）
任务：
1. 实现 LLM Pass-1 语义分块。
2. 实现 LLM Pass-2 标准行抽取。
3. 实现结构化规则校验层。
4. 实现 LLM Pass-3 疑难修复层。
5. 输出证据片段和置信度。

交付物：
1. 抽取引擎主流程。
2. 模块化提示词配置。
3. 低置信度复核输出。

DoD：
1. 核心字段可稳定产出。
2. 异常行不污染主表，能进入复核池。
3. 同样输入多次运行结果差异在可控范围内。

## 2.4 Package D：实验仪器订货表V1模板（P0）
任务：
1. 固定列映射与格式导出。
2. 人名误识别过滤与括号备注处理。
3. 多商品拆分与活动文案过滤。
4. 保留进价相关字段为空值策略。

交付物：
1. `order_sheet_lab_v1` 模块配置。
2. 对照样本回归结果。
3. 导出可交付样表。

DoD：
1. 与人工表核心字段匹配率达到阶段目标。
2. 日期统一 `YYYY-MM-DD`。
3. 常见误识别问题显著下降。

## 2.5 Package E：评测闭环与运营化（P1）
任务：
1. 人工修正回灌流程。
2. 样本检索增强（RAG few-shot）。
3. 周报指标看板（准确率、复核率、失败率）。
4. 异常类型自动聚类与优先级建议。

交付物：
1. 评测脚本与报表。
2. 运营复盘模板。
3. 持续优化SOP。

DoD：
1. 可持续追踪质量变化。
2. 闭环数据可用于下一轮模块优化。

## 3. 优先级与依赖
1. P0 先后顺序：A -> B -> C -> D。
2. E 在 D 达到基础准确率后并行推进。
3. A/B 未完成前，不进入大规模模板扩展。

## 4. 里程碑计划（建议）
1. M1（3-5天）：完成 A + B，打通可切换与进度可见。
2. M2（5-7天）：完成 C，跑通LLM主链路并可导出。
3. M3（3-5天）：完成 D，达到首轮客户验收标准。
4. M4（持续）：完成 E，进入持续优化运营。

## 5. 研发协作分工建议
1. 后端A组：任务层、模块层、导出层。
2. 后端B组：LLM抽取链路、规则校验、修复链路。
3. 前端组：导出页面、筛选交互、进度展示。
4. 测试组：对照验证、性能与异常测试。
5. 产品/运营：验收样本与复核规则确认。

## 6. 开工前检查清单
1. 文档版本冻结并同步全员。
2. 测试账号与租户权限准备完成。
3. 历史对照数据已放置并校验可读取。
4. 模型服务配额和重试策略确认。
5. 回归脚本可在本地与模拟云模式运行。

## 7. 风险追踪项
1. 模型调用耗时波动导致任务延迟。
2. 长消息批次拆分不合理导致抽取失真。
3. 客户样本口径变化导致匹配率波动。
4. 多模块并存时版本管理复杂度上升。

## 8. 完工定义（项目级）
1. P0 全部通过。
2. 关键测试项通过率达到门槛。
3. 管理员可独立完成模块切换。
4. 形成一套可复制到新客户的接入模板流程。
