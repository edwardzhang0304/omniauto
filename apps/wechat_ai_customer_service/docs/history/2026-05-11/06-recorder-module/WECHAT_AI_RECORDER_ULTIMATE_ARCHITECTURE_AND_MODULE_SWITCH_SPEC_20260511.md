# 微信AI智能记录员终极架构与模块切换规范（2026-05-11）

## 1. 设计原则
1. 底座能力稳定复用，客户差异通过模块扩展，不污染主流程。
2. LLM负责语义理解，规则负责边界校验和稳定兜底。
3. 导出链路可观测、可追溯、可重试。
4. 任何“客户定制”都必须可开关、可版本化、可回滚。

## 2. 分层架构
1. 采集层：监听微信会话，落原始消息库。
2. 任务层：创建结构化导出任务、排队调度、状态机驱动。
3. 抽取层：语义分块、字段抽取、后处理修复。
4. 质量层：一致性校验、置信度打分、复核池。
5. 导出层：模板映射、Excel/JSON输出。
6. 配置层：模块注册、账号绑定、灰度控制。

## 3. 运行主链路
1. 用户发起导出任务（可带日期范围/快捷周期）。
2. 任务服务按筛选条件拉取原始消息。
3. 抽取引擎执行四阶段：
- 阶段A：LLM语义分块（订单单元切分）。
- 阶段B：LLM结构化抽取（标准JSON输出）。
- 阶段C：结构化校验（日期、数量、金额、噪声过滤）。
- 阶段D：LLM二次复核（仅修复疑难行）。
4. 导出服务按模块模板映射字段并生成 Excel。
5. 结果落库并回传任务状态、下载地址、质量统计。

## 4. 模块化设计（关键）
### 4.1 模块组成
每个模块由四类 profile 组成：
1. `SchemaProfile`：目标字段定义、字段约束、默认值策略。
2. `ExtractionProfile`：LLM提示词模板、few-shot样本、候选过滤策略。
3. `PostProcessProfile`：规则校验、冲突处理、日期格式规范。
4. `ExportProfile`：Excel列顺序、列名、格式样式与导出后处理。

### 4.2 模块注册表示意
```json
{
  "module_id": "order_sheet_lab_v1",
  "module_name": "实验仪器订货表V1",
  "module_version": "1.0.0",
  "status": "active",
  "profiles": {
    "schema": "schema_order_sheet_lab_v1",
    "extraction": "extract_order_sheet_lab_v1",
    "post_process": "post_order_sheet_lab_v1",
    "export": "export_order_sheet_lab_v1"
  }
}
```

### 4.3 账号绑定规则
1. admin在服务端给账号绑定 `module_id`。
2. 运行时优先读取账号绑定模块。
3. 若账号未绑定，回落到租户默认模块。
4. 若模块不可用，回落到系统默认通用模块并记录告警。

## 5. 通用能力与专用能力边界
### 5.1 通用底座（必须稳定）
1. 采集入库、任务队列、状态可视化、日期筛选、导出框架。
2. 统一错误码、日志、重试、审计。
3. 模块注册和账号绑定。

### 5.2 专用模块（可替换）
1. 业务术语词典（产品别名、习惯话术）。
2. LLM提示词策略与few-shot样本。
3. 导出列定义与字段映射策略。
4. 模块级复核规则。

## 6. 状态机与进度可视化
### 6.1 状态定义
1. `queued`：已入队，等待执行。
2. `preprocessing`：消息筛选、清洗、分片。
3. `extracting`：LLM主抽取阶段。
4. `reviewing`：规则校验与LLM复核修复。
5. `exporting`：生成Excel和元数据。
6. `completed`：成功完成。
7. `failed`：失败并输出原因。

### 6.2 用户可见反馈
1. 当前阶段名。
2. 已处理条数/总条数。
3. 最近更新时间与预估剩余时长。
4. 异常时显示可执行动作：重试、下载中间结果、查看失败原因。

## 7. 可扩展性设计
1. 新客户接入仅新增模块，不改主任务编排。
2. 支持多模块共存和版本化升级。
3. 支持A/B模块灰度：账号组A使用 v1，账号组B使用 v2。
4. 支持模块级审计，记录每次导出的模块版本与参数快照。

## 8. 降级与回滚
1. LLM不可用时，允许降级到规则保底导出（并标注低可信）。
2. 模块升级失败时，按账号回退到上一个稳定版本。
3. 导出失败不影响采集入库链路。
4. 回滚不修改历史导出结果，只影响新任务。

## 9. 关键设计决策
1. 不把客户特例硬编码在通用服务中，避免后续不可维护。
2. 不依赖单一正则规则，避免泛化失败。
3. 强制保留证据片段，保证可解释和可复盘。
4. 抽取和导出解耦，便于后续支持更多表格形态。
