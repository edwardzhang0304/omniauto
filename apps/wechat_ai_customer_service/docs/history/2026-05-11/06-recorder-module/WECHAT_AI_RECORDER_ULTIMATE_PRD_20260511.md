# 微信AI智能记录员终极优化 PRD（2026-05-11）

## 1. 背景与目标
当前“AI智能记录员”已经具备导出能力，但在复杂自然语言场景中存在抽取不稳定、客户格式定制成本高、模块切换不够标准化的问题。
本次目标是构建“LLM主导理解 + 结构化辅助校验”的升级方案，并做到多客户模板可切换，支持按账号配置能力。

## 2. 产品目标
1. 能从微信群/个人聊天中智能识别订单语义，覆盖自然语言表达和新表达方式。
2. 能输出稳定结构化结果并导出 Excel，格式对齐客户订货表模板。
3. 保持底座通用，客户专用能力作为可切换模块，支持 admin 按账号绑定。
4. 在准确率达标前提下，提供可追溯、可复核、可持续优化机制。

## 3. 业务范围
### 3.1 In Scope（本期）
1. 微信聊天记录采集、入库、筛选。
2. 结构化导出任务创建、执行、状态可视化、结果下载。
3. 高级日期筛选（起止日期）和快捷导出（日/周/月）。
4. 模板化导出模块与账号绑定切换机制。
5. 实验仪器订货表V1模板（进价/总进价可空）。

### 3.2 Out of Scope（本期不做）
1. 图片/OCR信息抽取。
2. 产品库自动补齐进价逻辑（仅预留接口）。
3. 自动财务对账与自动下单。
4. 跨系统ERP写回。

## 4. 关键用户与角色
1. 租户管理员（admin）：配置账号、绑定模块、查看任务和导出结果。
2. 运营用户：发起导出、检查结果、处理复核。
3. 开发/运维：维护模块、监控任务、处理异常。

## 5. 核心用户故事
1. 作为运营用户，我希望选择日期范围后一键导出，这样我能快速拿到指定周期订货汇总。
2. 作为管理员，我希望按账号切换结构化模块，这样不同客户能复用同一底座。
3. 作为业务负责人，我希望导出结果可追溯原文证据，这样能快速判断抽取是否可信。
4. 作为交付团队，我希望同一框架下仅替换模板即可服务新客户，这样上线更快、维护更低。

## 6. 需求明细
### 6.1 功能需求（FR）
FR-001：记录员监听与入库
- 支持目标微信群/个人消息持续采集。
- 原始消息含时间、发送人、会话标识、内容、去重键。

FR-002：导出任务生命周期
- 状态包含：`queued`、`preprocessing`、`extracting`、`reviewing`、`exporting`、`completed`、`failed`。
- 前端自动轮询，无需手工“刷新状态”按钮。

FR-003：日期筛选导出
- 支持开始/结束日期自由选择。
- 支持快捷键：按日、按周、按月。
- 默认导出条数上限 10000，实际不足则全量导出。

FR-004：结构化抽取
- 采用“语义分块 -> 结构化抽取 -> 规则校验 -> LLM复核修复”。
- 输出标准行模型，含置信度与证据片段。

FR-005：模块切换
- 管理端可给账号绑定模块ID。
- 模块至少包含：字段模板、抽取策略模板、后处理模板、导出模板。
- 不同模块可并存，同一账号仅生效一个主模块（可允许灰度备用模块）。

FR-006：导出格式
- 输出 Excel 列顺序固定，对齐“订货表”期望格式。
- 无法可靠获取的字段允许空值，不得伪造。

FR-007：复核与追溯
- 低置信度行自动打标进入复核池。
- 行级可查看原文证据和抽取理由摘要。

### 6.2 非功能需求（NFR）
NFR-001：准确率目标
- 产品名准确率 >= 92%。
- 数量准确率 >= 95%。
- 日期完整率（YYYY-MM-DD）= 100%。

NFR-002：性能目标
- 1000条文本在正常模型配额下，10分钟内完成导出。
- 任务失败支持断点重试。

NFR-003：可维护性
- 新客户上线不改底座主流程，仅新增/替换模块配置。
- 所有模块有版本号，支持回滚。

NFR-004：可观测性
- 每个任务有阶段耗时、失败原因、重试次数、模型调用统计。

## 7. 成功指标（KPI）
1. 人工表 vs AI表：核心字段行级匹配率达到阶段目标（先 80%，后 90%+）。
2. 低置信度复核率随迭代下降。
3. 新客户模板接入周期从“改代码”降为“配模块 + 少量策略开发”。

## 8. 约束与假设
1. 数据源为文字消息，图片暂不纳入。
2. 聊天上下文可能缺失完整字段，允许部分字段空置并标记。
3. 模型服务可用性受外部配额影响，需要任务级重试与降级策略。

## 9. 风险与应对
1. 风险：自然语言变化快，规则易失效。
对策：LLM主导抽取 + 人工纠正样本回灌。
2. 风险：任务长时间“处理中”导致用户焦虑。
对策：阶段进度可视化 + 超时告警 + 失败可重试。
3. 风险：不同客户格式冲突。
对策：模块隔离、租户绑定、模板版本治理。

## 10. 里程碑
1. M1：底座能力补齐（任务状态、日期筛选、模块绑定、进度展示）。
2. M2：LLM主导抽取链路上线（含复核池）。
3. M3：实验仪器模板达标并形成可复制接入规范。

## 11. 验收结论模板
1. 功能验收：全部 FR 是否通过。
2. 指标准入：NFR 指标是否达标。
3. 风险残留：是否需要灰度限制。
4. 上线建议：可上线 / 灰度上线 / 延后上线。
