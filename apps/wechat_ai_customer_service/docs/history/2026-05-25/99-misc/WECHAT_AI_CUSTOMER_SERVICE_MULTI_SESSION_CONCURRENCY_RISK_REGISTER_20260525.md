# 微信自动客服多会话并发调度风险登记（2026-05-25）

## 1. 风险分级

| 等级 | 含义 |
|---|---|
| P0 | 会造成串会话、误发、重复发送、风控停机或数据损坏，必须阻断上线 |
| P1 | 会造成明显漏处理、长时间积压或不可恢复失败，必须修复后验收 |
| P2 | 影响体验或效率，但有可接受降级 |
| P3 | 观测、文案或运维便利性问题 |

## 2. 风险清单

### R1：RPA 并发操控微信

- 等级：P0
- 表现：多个线程同时切窗口、读消息或发送。
- 后果：误发、拖拽、白屏、风控风险。
- 控制：全局 RPA lock；LLM worker 禁止调用 RPA；send runner 单线程。
- 验收：测试中无并发 RPA 调用。

### R2：旧上下文回复误发

- 等级：P0
- 表现：客户在 LLM 思考期间补充消息，但旧回复仍发送。
- 后果：答非所问，用户体验差。
- 控制：context_version；发送前 freshness check；stale reply 阻断。
- 验收：补充消息场景旧回复发送数为 0。

### R3：活跃会话超过上限后 pending 丢失

- 等级：P0
- 表现：10 个会话同时发消息，本轮只处理 3 个，剩余会话下一轮被清掉。
- 后果：漏回。
- 控制：截断前写 pending；未处理 pending 不能因无变化清除。
- 验收：pending 超限测试中剩余会话全部保留。

### R4：会话列表 OCR 信号不足

- 等级：P1
- 表现：纯 RPA 只识别会话名，不能稳定识别预览、时间或未读角标。
- 后果：无法可靠发现新消息。
- 控制：增强 OCR 解析；缺少信号时不清 pending；配置白名单轮扫兜底。
- 验收：no-send 实盘能捕获多个测试会话。

### R5：LLM worker stuck

- 等级：P1
- 表现：任务长期 running，不释放同会话 inflight。
- 后果：该会话不再生成回复。
- 控制：timeout；heartbeat；stuck task recovery。
- 验收：超时任务自动 failed 或 fallback。

### R6：发送队列积压

- 等级：P1
- 表现：ready reply 不断增长但发送不出去。
- 后果：客户等待长、旧回复过期。
- 控制：backpressure；runtime status；reply TTL；人工告警。
- 验收：积压时系统显示黄色/告警，不静默。

### R7：发送失败重复重试

- 等级：P0
- 表现：RPA 发送结果不明确，系统重复发送。
- 后果：重复回复或风控风险。
- 控制：发送后目标验证；unknown 状态人工确认；重试次数上限。
- 验收：失败模拟不产生重复客户可见消息。

### R8：F8 暂停只停 UI 不停队列

- 等级：P0
- 表现：悬浮球显示暂停，但后台继续发送。
- 后果：用户无法安全接管。
- 控制：operator guard 状态作为 capture/send 派发硬门禁。
- 验收：暂停后无新 RPA capture/send。

### R9：进程重启导致队列丢失

- 等级：P1
- 表现：managed listener timeout 或电脑休眠后内存队列消失。
- 后果：漏处理或重复处理。
- 控制：持久化 scheduler state；启动恢复；processed ids 仍作最终去重。
- 验收：重启恢复测试通过。

### R10：高峰下过度 RPA 操作触发异常

- 等级：P1
- 表现：频繁切换会话、读取、发送导致微信白屏或风控信号。
- 后果：停机或被踢登录。
- 控制：RPA cooldown；低扰动扫描；被动掉线探针；异常立即停机。
- 验收：长测无白屏/掉线；异常检测能停机。

## 3. 降级策略

| 触发 | 降级 |
|---|---|
| LLM API 慢 | 降低并发，启用 fallback 或 handoff |
| 发送队列积压 | 暂停 capture 或只 capture no-send |
| RPA 窗口异常 | 停止 send，保留队列 |
| OCR 不可靠 | 白名单轮扫，禁止清 pending |
| 风控信号 | 立即停止 managed listener |
| state 损坏 | 备份损坏文件，重建空 scheduler state，保留旧 processed state |

## 4. 上线红线

以下任一未满足，不得进入真实客户长测：

- P0 风险均有测试覆盖。
- F8 暂停可阻断 capture 和 send。
- stale reply 不可发送有单元和集成测试。
- pending 超限不丢有单元测试。
- RPA-only 证明通过。
- 回滚开关验证通过。

## 5. 人工接管规则

应立即人工接管：

- 同一会话连续 2 次 send_failed。
- 任一 gap risk 无法恢复。
- 任一会话 pending 超过 10 分钟。
- 任一 transport risk stop。
- 任一疑似串会话发送。
- 任一微信白屏、登录窗口、输入框不可用。
